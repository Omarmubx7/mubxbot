# Cleanup Checklist

This file lists temporary files that should be deleted from the repository.

## Files to DELETE
These are temporary or auto-generated files that shouldn't be in version control:

- [ ] `update.txt` - Temporary instruction file
- [ ] `README_GENERATION_PROMPT.md` - Prompt used for README generation (not needed)
- [ ] `designdox.md` - Old design notes (consolidated elsewhere)
- [ ] `.git/COMMIT_EDITMSG` - Auto-generated git file (ignored by default)

## Files to KEEP
These are important and should stay:

- ✅ `README.md` - Main documentation (now human-friendly)
- ✅ `TECHNICAL.md` - Technical documentation
- ✅ `PERFORMANCE_OPTIMIZATIONS.md` - Performance guide
- ✅ `AI_SETUP.md` - AI/Azure setup
- ✅ `BUG_REPORT.md` - Bug reporting template
- ✅ `TEAMS_SETUP.md` - Teams integration guide
- ✅ `DOCS.md` - Documentation index
- ✅ `rules.md` - System rules

## How to Clean Up

### Via Git (Recommended)
```bash
# Remove temporary files
git rm update.txt README_GENERATION_PROMPT.md designdox.md

# Or just delete them manually, run:
git status  # to see what needs to be removed
```

### Via File Explorer
1. Delete these files directly in your file manager:
   - `update.txt`
   - `README_GENERATION_PROMPT.md`
   - `designdox.md`
2. Run: `git add -u` to stage the deletions
3. Commit: `git commit -m "cleanup: remove temporary files"`

**Note:** `.git/COMMIT_EDITMSG` is auto-generated by git and will disappear on its own.
